# Reader preferences

> This file describes **who you are and what you care about**. The LLM reads it
> to decide how relevant each item is. Edit it freely — no code changes needed.
> Be concrete: name specific topics, tools, and things you want to AVOID.

## About me
I'm a software engineer who wants to stay on top of AI. I care about both the
research frontier and practical engineering. I read in English (Japanese is fine
too). I have limited time, so I want signal over noise.

## I'm very interested in
- New, capable open-source / open-weight models (Llama, Qwen, Mistral, Gemma,
  DeepSeek, etc.) and how to run them locally.
- LLM inference, quantization, fine-tuning, RAG, agents, and tooling.
- Papers that introduce a genuinely new method, result, or benchmark — not
  incremental tweaks.
- High-quality engineering write-ups: real systems, lessons learned, benchmarks,
  architecture deep-dives written by practitioners.
- Notable model/product releases from major labs (Anthropic, OpenAI, Google
  DeepMind, Meta, Mistral, etc.).
- Developer tools and libraries that meaningfully improve AI workflows.

## I'm mildly interested in
- AI industry news with technical substance.
- Datasets and evaluation methodology.

## I want to avoid
- Pure hype, press releases with no technical content, and "AI will change
  everything" opinion pieces.
- Beginner tutorials and "what is an LLM" explainers.
- Marketing, funding announcements, and listicles.
- Low-effort Reddit posts (memes, "which GPU should I buy", help-me-debug threads).
- Duplicate coverage of a story I'd already see elsewhere.
